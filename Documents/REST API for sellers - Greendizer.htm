Collections

Listing email addresses

sellers/{seller id}/emails/
HEAD, GET Returns the list of emails attached to the buyer's account.

Listing, filtering and sending invoices

sellers/{seller id}/emails/{email id}/invoices/
HEAD, GET Returns the list of invoices sent by the user.
POST Sends one or multiple invoices to buyers.

You can filter Invoices based on the attributes in the invoice filters list.

sellers/{seller id}/emails/{email id}/invoices/?q=filter

To send a new invoice, make a POST request to this collection with a valid XMLi in its body.

POST sellers/{seller id}/emails/{email id}/invoices HTTP/1.1
Accept: application/json
Content-Type: application/xml

<?xml version='1.0' encoding='UTF-8'?>
<xmli version="gd-xmli-1.1">
    <invoices>
        <invoice>
            <buyer>
                <name>Jimi Hendrix</name>
                <email>jimi.hendrix@springshield.com</email>
                <address>
                    <streetAddress>15 rue de Chambery</streetAddress>
                    <zipcode>75015</zipcode>
                    <city>Paris</city>
                    <country>FR</country>
                </address>
            </buyer>
            <name>Name of the invoice</name>
            <description>Description of the content of the invoice</description>
            <date>2011-02-01</date>
            <dueDate>2011-02-25</dueDate>
            <currency>USD</currency>
            <total>100</total>
            <status>due</status>
            <body xmlns:myCompany="http://domain.example.com">
                <groups>
                    <group>
                        <name>Main group</name>
                        <description>Main Invoice Group</description>
                        <lines>
                            <line>
                                <name>Product or service name</name>
                                <description>Product description</description>
                                <quantity>10</quantity>
                                <unitPrice>1</unitPrice>
                            </line>
                        </lines>
                        <custom>
                        	<myCompany:points>343</myCompany:points>
                        </custom>
                    </group>
                </groups>
                <custom>
                	<myCompany:client>12545343</myCompany:client>
                </custom>
            </body>
        </invoice>
    </invoices>
<xmli>

Please refer to the XMLi reference for more information about how to build a valid XML invoice.

Listing and filtering invoicing reports

sellers/{seller id}/emails/{email id}/invoices/reports/
HEAD, GET Returns the list of invoice delivery reports for the seller.

You can also filter the invoicing reports using the attributes in the invoicing report filters list.

sellers/{seller id}/emails/{email id}/invoices/reports/?q=filter

Listing, filtering and opening new conversation threads

sellers/{seller id}/threads/
HEAD, GET Returns the list of threads.
POST Opens a new conversation thread with a user.

You can also filter the threads collection using the attributes in the thread filters list.

sellers/{seller id}/threads/?q=filter

To open a new conversation thread, make a POST request to this collection and include the following arguments in its body:

recipient The ID of the user with whom this thread will be opened as a string.
subject The subject of the thread as a string.
message The content of your first message as a string. 
POST sellers/0/threads HTTP/1.1
Accept: application/json
Content-Type: application/x-www-form-urlencoded

recipient=1284&subject=The wind&message=It cries Mary

Listing and sending messages

sellers/{seller id}/threads/{thread id}/messages/
HEAD, GET Returns the list of messages contained in a thread.
POST Creates a new message.

To send a new message, make a POST request to this collection and include the following arguments in its body:

text The content of your message as a string. 
POST sellers/0/threads/124/messages HTTP/1.1
Accept: application/json
Content-Type: application/x-www-form-urlencoded

text=Hello, world!

Seller

Overview

Represents the profile of a seller.

sellers/{seller id}/
Allowed methods HEAD, GET
Protected True

Attributes

firstname The first name of the seller as a string.
lastname The last name of the seller as a string.
birthday A timestamp in milliseconds representing the seller's birthday.
avatar The URI of the seller's avatar picture.
gender The gender of the seller.

Settings

Overview

Represents the settings of the seller.

sellers/{seller id}/settings/
Allowed methods HEAD, GET
Protected True

Attributes

language The language of the user as a two letter code string.
region The regional settings of the user as a string. The format is:
{two letter language code}-{two letter country code}.
currency A three digit representation of the currency set in the user's settings.

Company

Overview

Represents the company for whom the seller works.

sellers/{seller id}/company/
Allowed methods HEAD, GET
Protected True

Attributes

name The name of the company as a string.
description A short description of the company as a string.
verified A boolean indicating whether or not the company was verified.
smallLogo The URI of the small version of the company's logo as a string.
largeLogo The URI of the large version of the company's logo as a string.

Email

Overview

Represents an email address registered by a user or shared with him.

sellers/{seller id}/emails/{email id}
Allowed methods HEAD, GET
Protected True

Attributes

label An friendly name given by the user to refer to the email address as a string.
address The email address.
verified A boolean value representing the status of the email.

Buyer

Overview

sellers/{seller id}/buyers/{buyer email id}
Allowed methods HEAD, GET
Protected True

name The name of the buyer.
address An object representing the postal address of the buyer.
delivery An object representing the delivery address of the buyer if available.
currencies A list of objects containing statistics about the invoices on a currency basis.

threadsCount An integer representing the total number of conversation threads opened between the buyer and the current seller.
messagesCount An integer representing the total number of messages exchanged with between the buyer and the current seller.

Invoice

Overview

Represents an invoice sent by the seller.

sellers/{seller id}/emails/{email id}/invoices/{invoice id}
Allowed methods HEAD, GET, PATCH, DELETE
Protected True
OAuth scopes read_invoices, modify_invoices

Attributes

secretKey Read-only A random secret key generated for this invoice as astring.
name Read-only The title given by the seller to the invoice as a string.
description Read-only The description given by the seller to the invoice as a string.
buyer Read-only An object representing the buyer information included in the invoice.
  • name: the buyer name
  • email: the buyer email address
  • address: the buyer postal address
  • delivery: the buyer delivery address (if available)
  • uri: the buyer resource URI
date Read-only A timestamp representing the invoice date.
due_date Read-only A timestamp representing the date on which the invoice is due.
currency Read-only The currency of the invoice as a 3 letters code string.
total Read-only The total of the invoice as a float.
location An integer representing the folder in which the invoice is located:
  • 0 for Inbox
  • 1 for Archives
  • 2 for Trash
read A boolean indicating whether or not the invoice has been read.
flagged A boolean indicating whether or not the invoice has been flagged.
paid A boolean indicating whether or not the invoice has been paid.
payable Read-only A boolean indicating whether or not the invoice can be paid through Greendizer.
canceled Read-only A boolean indication whether or not the invoice has been canceled.
body Read-only The XML representation of the invoice body as a string.

Filters

flagged Boolean
read Boolean
paid Boolean
canceled Boolean
location Integer
date Timestamp
dueDate Timestamp
modifiedDate Timestamp
customId String The value of the customId field in the invoice.
currency String
secret String The unique secret ID of the invoice
total Float
buyer String The Buyer's email address

Invoicing Report

Overview

Represents a report with information about an invoice sending request.

sellers/{seller id}/emails/{email id}/invoices/reports/{report id}
Allowed methods HEAD, GET
Protected True
OAuth scopes modify_invoices

Attributes

startTime Date and time at which the server started processing your request as a timestamp.
elapsedTime Date and time at which the server finished processing your request as a timestamp.
state An integer representing the folder in which the invoice is located:
  • 0 for Accepted
  • 1 for Running
  • 2 for Postponed
  • 3 for Canceled
  • 4 for Succeeded
  • 5 for Failed
errorStack A string containing an error message if one was encountered.
invoicesCount A integer representing the number of invoices successfully processed.
hash An SHA-1 hash of the invoices you have sent as a string.
ipAddress The IP address of the machine that sent the invoice.

Filters

state Integer
hash String
sender String The email address of the buyer the invoice was sent to

Thread

Overview

Represents a thread which contains one or more messages.

sellers/{seller id}/threads/{thread id}
Allowed methods HEAD, GET, PATCH, DELETE
Protected True
OAuth scopes read_messages, modify_messages

Attributes

subject Read-only A string representing the subject of the thread.
snippet Read-only A snippet of the last message sent or received as a string.
lastMessage Read-only A timestamp of the last message sent or received in the thread.
sellerURI Read-only A string representing the URI of the seller.
location An integer representing the folder in which the thread is located:
  • 0 for Inbox
  • 1 for Archives
  • 2 for Trash
count An integer indicating the number of messages in the thread.
read A boolean indicating whether the thread has been read or not.
flagged A boolean indicating whether the thread has been flagged or not.

Filters

flagged Boolean
read Boolean
location Integer
date Timestamp
modifiedDate Timestamp The date the thread was last modified.
lastMessageDate Timestamp The date of the last message in the thread
buyer String The Buyer's email address

Message

Overview

Represents a message which belongs to a thread.

sellers/{seller id}/threads/{thread id}/messages/{message id}
Allowed methods HEAD, GET
Protected True
OAuth scopes read_messages, modify_messages

Attributes

text The text of the message as a string.
createdDate A timestamp representing the date at which the message was sent.
buyerURI [For Received Messages] The resource URI of the buyer sending the message.
